Flowmind — Design Grammar v0.1

The Shame-Free Design System

Every color, motion, word, and interaction in Flowmind has one job: keep the cortisol-shame pathway from activating. This is the canonical grammar for how.

Foundations

Seven Core Principles

Every design decision is measured against these. If a proposed element violates one, it goes back. No exceptions.

01

Effort is always honored

The system sees every micro-step completed. Nothing a user does inside Flowmind goes unacknowledged.

02

Absence is never punished

If a user hasn't opened the app in 3 days, their return is a welcome, not a reckoning. No streak-loss alerts. No red numbers.

03

The system does not raise its voice

No red. No bold warnings. No exclamation marks on negative states. Urgency is communicated through tone, never alarm.

04

Language separates behavior from identity

"This task wasn't started" — not "you missed this." Tasks have states. Users have worth. Never conflate them.

05

Everything is undoable

Mistakes are cheap. Deletion is reversible. No destructive action without a calm, easy escape. Fear of error causes paralysis.

06

Ambient calm over reactive urgency

The UI rests at a calm baseline. Joy and energy rise contextually. The resting state should feel like a quiet room, not a dashboard.

07

Celebrate small, celebrate often

Dopamine doesn't wait for big wins. Every micro-step gets acknowledgment — proportional, genuine, not patronizing.

Visual Language

Color System

The palette is built on warm organic neutrals rather than the cold clinical whites and saturated primaries of productivity tools. Every hue was chosen to reduce cortisol, not increase it.

Rule: Red is never used in user-facing negative states. Orange is used only at low saturation. The color language of failure and urgency is entirely absent.

Visual Language

Typography

The type system pairs a warm, humanist serif for display moments with a clean, legible sans for body. Never use condensed, angular, or high-contrast typefaces — they read as tense and demanding.

Display — Instrument Serif / Italic — Emotions, headlines, empty states, celebration

You did something.

That matters.

UI Body — DM Sans / 300–500 weight — All interface text

Send me an email about the thing by Friday

Flowmind turned this into 4 tiny steps. You only need to do one right now.

Any time today works. There's no wrong order.

Mono — DM Mono / 400 weight — Labels, metadata, timestamps, code tokens

Step 2 of 4  ·  ⭐ 3 stars available

Type Scale

32–40px Section Heading
20–24px Card title or key moment
15–17px Body text, task text, input fields
13px Supporting text, hints, secondary labels
10–11px Metadata / Tag labels / System labels
Rule: Maximum font weight in the UI is 500 (medium). Never use bold (700+) for negative states, warnings, or incomplete tasks. Bold draws attention — in Flowmind, bold is only used for celebration or core actions.

Interaction Design

Motion Grammar

Motion in Flowmind is calm, organic, and intentional. Fast snappy transitions feel demanding. Slow dragging transitions feel lethargic. The sweet spot signals care — the app responds to you, not at you.

Rule: Nothing in the UI vibrates, flashes, pulses, or bounces to express urgency or failure. Celebratory motion (reward earned, task done) is the only context for bouncy or energetic animation.

What we never animate

❌ Banned Motion Patterns

Red badge pulsing to indicate overdue tasks. Screen shake on error. Progress bar decreasing. Blinking reminders. Countdown timers with urgency styling. Loading spinners without calming language. Streak counter exploding on loss.

✓ Approved Motion Patterns

Gentle fade-in on new content. Soft scale-pop on celebration. Smooth ease-out progress fills. Items completing with a slide-and-fade. Stars floating up on reward. Screen dimming calmly when focus mode activates.

Content Strategy

Voice & Copy Rules

The language Flowmind uses is one of the most powerful levers on the shame pathway. Every string in the app should pass three tests: Does it separate the task's state from the user's identity? Does it carry zero implicit blame? Does it offer a door, not a wall?

The Flowmind voice is: a thoughtful friend who happens to have a PhD in cognitive neuroscience. Warm, informed, never clinical. Direct, never demanding. Honest, never harsh.
🚫 Don't say this ✓ Say this instead Why it matters
"You missed this task." "This one wasn't started yet." Separates task state from identity. The task has a state; you have worth.
"⚠️ Overdue! 3 tasks late." "3 things are waiting when you're ready." Removes urgency framing and alarm language. "Waiting" implies patience, not judgment.
"You haven't taken your meds." "Your meds reminder — just in case." "You haven't" is accusatory. "Just in case" is collaborative.
"Streak broken! You lost your 7-day streak." "Welcome back. Everything's here." Streak-loss language is specifically toxic for ADHD users. It restarts the shame cycle.
"Get back on track." "Pick up wherever feels easiest." "Back on track" implies you were off-track — a failure framing. Flowmind doesn't define a 'track.'
"You only have 2 hours left today!" "A couple hours of day left if you want them." Urgency language triggers cortisol. Optionality language keeps the user in choice, not panic.
"Nothing completed today." "Ready for you whenever you are." Counting zeros is a shame trigger. A waiting, patient presence is not.
"Try to be more consistent." Don't say this at all. Flowmind never prescribes behavioral norms or implies the user should be different.
"Error: something went wrong." "That didn't go through — want to try again?" Neutral framing + optionality + no blame. Errors are not catastrophes.
"You're doing great! 🎉🎉🎉" "That's done. Nice." Hollow cheerleading feels patronizing to adults. Brief, genuine acknowledgment lands better.

What to Avoid

Anti-patterns

These are the specific design patterns that other apps use which directly activate the shame pathway. They are banned in Flowmind — even when the team is tempted by their familiarity.

❌ Streak Mechanics

Duolingo-style streak counters

Any system that makes absence visible as a loss — broken chains, lost flames, zero-day markers — punishes the exact behavior pattern ADHD creates. When users return, they are already in shame. Do not add to it.

✓ Instead

Cumulative momentum

Show total stars earned (not lost). Show "most productive morning" not "days since last active." Celebrate the return, not the consistency. Momentum is additive, not subtractive.

❌ Red Badges & Alert Counts

The "12 overdue" red badge

App badges with high numbers, red indicators, and overdue counts are a cortisol delivery mechanism. ADHD users often avoid opening apps specifically to avoid seeing these. The app becomes a source of dread.

✓ Instead

Neutral or no badge

If a badge must exist, use a soft neutral color and a low number (or none). Lead with positive framing: "3 things ready for you" not "3 overdue."

❌ Countdown Timers

Pomodoro / deadline timers

Time pressure is a shame accelerant for ADHD. Countdown timers create urgency that often triggers freeze responses. Time blindness is a symptom, not a moral failure — designing around it means removing time as a punitive force.

✓ Instead

Time anchors, not timers

Show "this usually takes about 10 minutes" as ambient context. Offer optional focus sessions with count-up timers (showing effort, not countdown). Hard deadlines shown without panic formatting.

❌ Productivity Metrics

Daily score, completion rates, weekly averages

Dashboards showing productivity percentages, completion rates, or comparison to past performance transform the app into a measuring tool — and ADHD users will always feel measured against neurotypical norms.

✓ Instead

Moments and milestones

Celebrate specific wins ("You finished a thing you'd been avoiding for 2 weeks"). Show stars earned, not tasks missed. Let the user discover patterns themselves rather than having the app define them as a deficit.

❌ Prescriptive Scheduling

"You should do this at 9am"

Any feature that creates externally-imposed structure the user didn't define becomes something to fail against. Fixed schedules presented as the right answer frame the user's different rhythm as wrong.

✓ Instead

User-authored structure

Suggest, never prescribe. "Some people find morning easier for this — want to set a reminder?" The user defines the structure; Flowmind helps hold it. The locus of control stays with the user.

Mobile Experience

Haptic Language

Haptics are physical feedback — they bypass the visual cortex and go straight to felt experience. Flowmind uses haptics as a gentle, embodied communication layer. Never jarring, never punitive.

Medium pulse — step completed
Double pulse — task fully done
Triple soft — reward earned
Single light — reminder delivered
Soft single — navigation, confirmation
None — errors, missed tasks, overdue states
Rule: Errors and missed tasks produce zero haptic feedback. Haptics are celebratory or informational only. Vibration should never be associated with a negative outcome.
