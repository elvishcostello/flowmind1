# Flowmind Brand Guidelines
Version 1.0 · April 9, 2026

---

## Mission & Voice

Flowmind is the first neuroscience-backed, AI-first ADHD app that works *with* the neurodivergent brain, not against it. Every design decision should reinforce a single feeling: **relief, not obligation**.

The brand has four personality pillars:

- **Understanding** — We get it because we live it
- **Calming** — Minimalist design that reduces cognitive load
- **Empowering** — Progress over perfection, always
- **Playful** — Gentle nudges and rewards, never shame

## Colors

| Token | Hex | Usage |
|---|---|---|
| Brand Blue | `#011635` | Headers, primary text, borders, icons |
| Brand Pink | `#EED8FE` | Accents, badges, highlights, rewards |
| Page background | `#F3EEF8` | Lavender-tinted — reduces harshness vs. pure white |
| Surface / card | `#FFFFFF` | Content areas, modals, cards |
| Muted text | `#4A5568` | Body copy, captions, secondary labels |
| Dividers | `#D8CCE8` | Rules, separators |
| Pink deep | `#C9A8F0` | Highlight borders, active states |

**Usage rules:**
- Brand Blue is the dominant color — headers, CTAs, navigation
- Brand Pink is an accent only — never use it as a large background block
- Maintain generous white or lavender space; visual clutter directly increases cognitive load for the target audience
- Never use color as the sole indicator of meaning (accessibility requirement)
- All text must meet **WCAG AA contrast** (4.5:1 minimum for body text)

## Typography

### Brand Fonts (licensed — commercial)
| Role | Font | Source |
|---|---|---|
| Display / serif | Equity Sans | creativemarket.com |
| Body / sans | MADE Tommy Soft | creativemarket.com/MadeType/4689694 |

MADE Tommy Soft has multiple weights (ExtraLight through Black, plus Outline variants). Prefer **Regular** and **Medium** for body; **Bold** only for emphasis or CTAs.

### Web-safe substitutes (in use until licensed fonts are installed)
| Role | Substitute | Notes |
|---|---|---|
| Display / serif | Cormorant Garamond | Google Fonts — closest freely available serif |
| Body / sans | Nunito | Google Fonts — rounded, friendly, low cognitive load |

### Type scale
| Level | Size | Usage |
|---|---|---|
| Brand name | 1.75rem italic | Page/screen headers only |
| Doc label | 0.875rem uppercase, tracked | Sub-headers, category labels |
| Section heading | 1.375rem | Section titles (paired with numbered badge) |
| Body | 1rem | All paragraph and list text |
| Caption / meta | 0.75rem | Dates, footnotes, legal fine print |

## Spacing & Layout

- Max content width: **680px** (comfortable reading line length)
- Base padding: **1.5rem** sides, **2.5rem** top, **4rem** bottom
- Section spacing: **2.5rem** between sections
- Line height: **1.75–1.8** for body text — wider than standard to ease reading
- All spacing defined on an `xs / sm / md / lg / xl` scale for consistency

## Components

### Numbered section badge
A small pill in Brand Pink (`#EED8FE`) with Brand Blue text. Used to label ordered sections (01, 02 …). Reinforces calm hierarchy without using heavy borders or lines.

### Highlight block
Left-border callout in Brand Pink with a deeper pink (`#C9A8F0`) border. Used for key statements the user must not miss (e.g. "we do not sell your data"). Never use red or warning colors — the tone must remain calm and trustworthy.

### Dividers
`1px` rule in `#D8CCE8` for internal separators; `2px` Brand Blue rule under the main page header only.

---

## Tone & Language

- Short sentences. No legal-sounding jargon where plain language works.
- Shame-free copy throughout — no language that implies the user has done something wrong or failed.
- Celebratory framing where possible ("You're all set" not "Submission received").
- Labels and microcopy should feel like a helpful friend, not a system message.

---

## What to Avoid

- Pure white backgrounds — always use the lavender tint (`#F3EEF8`) or white surface cards on lavender
- Red, orange, or high-alert colors — they trigger stress responses in the target audience
- Dense text blocks or tight line spacing — increase cognitive load
- Emoji as primary UI indicators — use sparingly and never as the sole conveyor of meaning
- Perfectionist framing in any copy ("complete," "finish," "must") — prefer "start," "try," "add"
